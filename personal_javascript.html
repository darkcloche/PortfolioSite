Sam Howels - Designer

Top Down Input and Movement Investigation

Javascript Learning Exercise

After spending some time learning the basics of Javascript, I wanted to put my programming optimism to the test and use my new knowledge on a tangible project. Around the same time there was a lot of discussion in the office on how input, interaction and movement are an important (and sometimes overlooked) part of differentiating a game and giving it its own feel. I decided to put both of these interests together and create a small top down prototype in Javascript (built on the Phaser HTML5 framework) - the playable result of this can be found by clicking the image below. I had a few areas of focus for the core interaction:

  • Making the character movement feel like it has some weight whilst still retaining the responsiveness to feel satisfying to control.
  • Presenting button prompts in a discreet yet functional way. I wanted to clearly demonstrate the effect of a button press while having the prompts themselves feel like an extension of the game objects themselves as opposed to feeling presented by the game interface.
  • Experimenting with communicating interactive elements within the game in simple yet readable ways. I settled on a use-radius system that doubled as the language for interaction, with a fading system that indicates usability but requires the player to explore the environment in order to locate them.
