Sam Howels - Designer

Top Down Input and Movement Investigation

Javascript Learning Exercise

After spending some time learning the basics of Javascript, I wanted to put my programming optimism to the test and use my newfound knowledge on a tangible project. Around this time there was a lot of discussion in the office on how input, interaction and core movement mechanics on a base level are a very important (and sometimes overlooked) part of differentiating a game and giving it its own "feel". It can often be easy for developers to implement the "default" genre implementation of movement and interaction mechanics, which are then functional but are a missed opportunity for the game's concept to potentially manifest itself in original and interesting ways within the core gameplay.

I decided to put both of these interests together and create a small top down prototype in Javascript (built on the Phaser HTML5 framework). Not only did this give me a sensibly sized project to start applying object oriented programming concepts, but it also meant I could build core movement and interaction systems from the ground up and experiment with how to make these feel satisfying at a very base 2D level. I had a few areas of focus for the core interaction:

  • I wanted to make the movement feel non-binary and "weighty" to add some life to the player character, whilst retaining enough input responsiveness that the play experience isn't frustrating.
  • I wanted to present input prompts in a discrete yet functional way. My aim was to clearly demonstrate the effect of a button press when interacted with, while having the prompts themselves appear to be somewhat of an extension of the game objects themselves as opposed to feeling like game hints that are abstracted from the game world.
  • I wanted to communicate interactive objects in a simple yet readable fashion. I settled on a use-radius system that doubled as the language for interaction, as the radius fades in based on proximity. This provides clear feedback to players of whether an object can be interacted with, but as its not instantly visible it requires players to explore the environment to locate them.
